TWR-LM 현재 정리 및 실험 메모

이 문서는 현재 저장소 상태를 기준으로 다시 정리한 간단한 실행 메모다.
예전의 write-think-read slot memory 설계 문서는 더 이상 메인라인 구현과 일치하지 않는다.

1. 현재 메인라인 목표

- Transformer를 대체할 수 있는 tokenless latent backbone을 만든다.
- token-wise persistent hidden state를 backbone 상태로 유지하지 않는다.
- overlapping latent compression 이후에는 latent sequence만으로 계산한다.
- 파라미터 효율과 재현 가능한 benchmark 비교를 우선한다.

2. 현재 메인라인 구조

tokens
-> overlapping latent encoder
-> latent sequence
-> shared refinement
-> selective pairwise merge
-> multiscale query readout
-> logits

핵심 파일
- src/twr/models/twr_lm.py
- src/twr/modules/latent_encoder.py
- src/twr/modules/latent_backbone.py

3. 현재 비교 대상

- TWR lean
- Transformer
- Mamba

4. 현재 벤치마크

- LRA ListOps
- RULER needle
- LongBench TREC
- Hyperpartisan

5. 최신 결과 요약

ListOps
- TWR lean: 0.2285, 82,587 params
- Transformer: 0.2383, 86,026 params
- Mamba: 0.1289, 95,530 params

RULER needle
- TWR lean: 0.0840, 100,065 params
- Transformer: 0.0801, 117,136 params
- Mamba: 0.0781, 252,048 params

LongBench TREC
- TWR lean: 0.1800, 494,083 params
- Transformer: 0.1800, 627,250 params
- Mamba: 0.1800, 1,272,242 params

Hyperpartisan
- TWR lean: 0.5400, 475,283 params
- Transformer: 0.5450, 624,130 params
- Mamba: 0.5150, 1,266,050 params

6. 현재 해석

- Transformer가 아직 전체적으로 가장 안정적이다.
- TWR lean은 ListOps, Hyperpartisan에서 Transformer에 거의 붙었다.
- TWR lean은 RULER에서 근소 우세를 유지한다.
- TWR lean은 LongBench TREC에서 Transformer와 동률이다.
- 이번 설정에서는 Mamba가 네 벤치 모두에서 우세하지 않다.

7. 현재 우선순위

- 긴 문맥 태스크에서 lean TWR의 성능 저하를 줄일 것
- 파라미터를 크게 늘리지 않고 long-context headroom을 회복할 것
- text-lean과 long-lean 설정을 더 분리해서 최적화할 것
